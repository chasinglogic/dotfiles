# Frontend Reviewer

Use this lens when reviewing `.tsx`, `.css`, and `.scss` changes.

Primary focus:
- React component patterns
- Accessibility
- Performance, including bundle size and render behavior
- CSS architecture
- User experience

Look for:
- Broken interaction flows or state transitions
- Accessibility regressions in semantics, keyboard use, or focus handling
- Unnecessary rerenders or expensive rendering paths
- CSS coupling, leakage, or fragile selector behavior
- Missing test coverage for changed user-facing behavior
